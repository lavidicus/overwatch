## Lessons Learned

### Context Management
1. **Always check compaction settings** when debugging context issues - missing `reserveTokens` causes 100% trigger
2. **Pre-compaction flush is essential** for long-running sessions - preserves decisions and context
3. **4k token buffer** between flush and compaction provides safe headroom
4. **Layered approach works best**: flush (42k) + compaction (46k) + reserve (20k) + context-shift
5. **Memory files are valuable** - they become the session's long-term memory

### Configuration Management
1. **Document all config changes** in ITIL system for audit trail
2. **Use change requests** (CR-XXX format) for tracking
3. **Push to GitHub** after every change for version control
4. **Update CHANGE_TICKETS.md** to maintain overview

### Technical Notes
- ollama `--context-shift` flag is a good complement to compaction settings
- 66k context window with 20k reserve = 70% trigger point
- Memory flush runs silently with NO_REPLY - user sees no interruption
- Gateway restart required for config changes to take effect
