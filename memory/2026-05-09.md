## Session: 02:58 UTC — MTP Performance Tuning + USM1 Storage Setup

### MTP Speculative Drafting Optimization (pve3090-111)
- **Problem:** llama.cpp on pve3090-111 running Qwen3.6-27B-Q8_0 with MTP showed low generation speed (40.7 tok/s) and poor draft acceptance rate (45.3%) with `--spec-draft-n-max 5`
- **Root cause:** 5 draft tokens per step was too aggressive — drafts 4 and 5 were mostly low-confidence predictions getting rejected, burning GPU cycles
- **Fix:** Changed `--spec-draft-n-max 5` → `--spec-draft-n-max 3` in `/etc/systemd/system/llama-server.service`
- **Result:** Acceptance rate improved to 78-100%, generation speed improved to 51-65 tok/s, draft gen time dropped from 4077ms → 391-629ms
- **Service file path:** `/etc/systemd/system/llama-server.service` on pve3090-111
