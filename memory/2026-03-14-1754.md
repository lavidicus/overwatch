## 🔧 Root Cause & Fix

**Problem:** The gateway's HTTP `/tools/invoke` endpoint has a **default deny list** that blocks `cron`. AntFarm calls this endpoint to create cron jobs for agent polling.

**Wrong approach:** `gateway.scopes: ["operator.read"]` — invalid key, rejected by config schema.

**Correct fix:** 
```json
"gateway": {
  "tools": {
    "allow": ["cron"]
  }
}
```

This removes `cron` from the default HTTP deny list, letting AntFarm create cron jobs via `/tools/invoke`.
