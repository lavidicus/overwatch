```bash
#!/usr/bin/env bash
set -euo pipefail

if [[ $# -ne 1 ]]; then
  echo "Usage: $(basename "$0") <URL-to-GGUF>" >&2
  exit 1
fi

URL="$1"
FILENAME=$(basename "${URL%%\?*}")

# Ensure output directory exists
mkdir -p /opt/models/gguf

# Prefer wget, fallback to curl
if command -v wget >/dev/null 2>&1; then
  wget -O "/opt/models/gguf/${FILENAME}" "$URL"
elif command -v curl >/dev/null 2>&1; then
  curl -L -o "/opt/models/gguf/${FILENAME}" "$URL"
else
  echo "Error: wget or curl is required." >&2
  exit 2
fi

echo "Downloaded to /opt/models/gguf/${FILENAME}"
```

### Recommended baseline flags
- `--model` (required)
- `--port` (bind port)
- `--ctx-size` (8k–16k typical)
- `--n-gpu-layers` (set high; 3090 can usually take all layers for 7B/13B; use `-1` for auto)
- `--threads` (CPU threads for prompt processing; usually `nproc`)
- `--batch-size` (256–1024 depending on VRAM/model; 512 is safe start)
- `--parallel` (1–4 depending on concurrency)
- `--host 0.0.0.0` (if you need LAN access)
- `--no-mmap` only if storage is slow (usually leave mmap on)

### Example ExecStart (safe + fast)
```bash
ExecStart=/opt/llama.cpp/build/bin/llama-server \
  --model /opt/models/gguf/MODEL.gguf \
  --host 0.0.0.0 \
  --port 8080 \
  --ctx-size 8192 \
  --n-gpu-layers -1 \
  --threads $(nproc) \
  --batch-size 512 \
  --parallel 2
```

### If you want max GPU use (typical 7B/13B)
```bash
--n-gpu-layers -1 --batch-size 1024 --parallel 2
```

### If you want lower VRAM pressure

