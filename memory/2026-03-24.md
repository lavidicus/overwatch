# Matrix E2EE Media Fix — 2026-03-24 Session

## Goal
Resolve persistent Matrix E2EE image attachment failures where messages from `@lavid:comms.9xc.io` show `[matrix image attachment unavailable]` to Sam's `OPENCLAW_GW` device.

## Root Cause
**ES module bundling bug in OpenClaw 2026.3.23-x:** The `@matrix-org/matrix-sdk-crypto-nodejs` native module loader uses `__dirname` to find the `.node` binary, but `__dirname` is undefined in ES modules.

### Error Chain
1. `__dirname is not defined in ES module scope` → crypto loader fails
2. Falls back to `require("@matrix-org/matrix-sdk-crypto-nodejs-linux-x64-gnu")` → module not found
3. `matrix media download failed` in logs
4. Text messages work (no decryption needed), images fail (require crypto module)
