# 2026-03-09 - Tag Team Response Loop Issue

## Issue
Both bots (Sam and Eve) responded when user directly messaged the other, causing a response loop.

**Symptoms:**
- User @Eve → Sam also responds
- User @Sam → Eve also responds
- Bots get into echo loop saying "Everything's good"

## Root Cause
- Both bots are Signal group members and see all messages
- Each bot checks if addressed by name, but the trigger detection might not be strict enough
- When one bot responds, the other sees it and might interpret as needing to respond

## Fix Applied
Updated MEMORY.md with stricter addressing rule:

```
- Sam only responds when @Sam appears (case-sensitive, WITH @ symbol)
- Eve only responds when @Eve appears (case-sensitive, WITH @ symbol)
- CRITICAL: If you see the other bot responding to a message you didn't see addressed, STAY SILENT
```

## Key Constraint
**DO NOT echo, repeat, or acknowledge the other bot's messages.** Only respond when directly addressed with @Sam or @Eve.

## Action Items
- [ ] Verify both bots have this constraint in their memory
- [ ] Test direct messaging to each bot separately
- [ ] Confirm bots don't respond to each other's messages
