## N8n Deep Debug & Fix (17:30 - 18:02 UTC)

### The Problem
Qwen3.5 kept "waiting" for webhook to respond, but webhook always returned:
```
{"code":404,"message":"The requested webhook \"POST sam-trigger\" is not registered"}
```

### Root Causes Found (3 Critical Bugs)

**1. Missing `webhookId` field (CRITICAL)**
- N8n v2 requires `webhookId` on webhook nodes
- Without it, webhook NEVER registers even if workflow shows `active: true`
- Fix: Added `"webhookId": "sam-trigger-id"` to webhook node

**2. `child_process` blocked in Code nodes (SANDBOX)**
- N8n's task runner sandboxes Code nodes for security
- `require('child_process')` throws: `Module 'child_process' is disallowed [line 1]`
- Fix: Created HTTP bridge service to wrap CLI tools

**3. IPv6 localhost mismatch (NETWORK)**
- N8n resolves `localhost` to IPv6 `::1`
- Bridge service listens on IPv4 `127.0.0.1:18790`
- Fix: Use `http://127.0.0.1:18790` in HTTP Request nodes (not `localhost`)

### The Fix: Gog Bridge Service

Created a local HTTP server to proxy CLI commands:
- **Service:** `gog-bridge.service` on port 18790
- **Script:** `/home/localadmin/.n8n/scripts/gog-bridge.js`
- **Endpoint:** `http://127.0.0.1:18790/fetch-emails`
- **Runs:** `/home/linuxbrew/.linuxbrew/bin/gog gmail messages search ...`
- **Systemd:** `sudo systemctl {start|stop|restart} gog-bridge`
