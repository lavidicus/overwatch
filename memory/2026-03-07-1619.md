## Template Created

**Location:** `ITIL/playbooks/TEMPLATE.md`

### Structure

```
PLAYBOOK TITLE
├── Overview (2-3 sentences)
├── Priority (P1/P2/P3 + justification)
├── Category (Incident/Change/Ops)
├── Prerequisites (access, tools, info, dependencies)
├── Status Checks (pre-procedure verification)
├── Procedure (numbered steps with commands + expected output)
├── Verification (confirm success)
├── Common Issues (symptoms → diagnosis → resolution)
├── Rollback (undo procedure)
├── Related Playbooks (cross-references)
└── Notes (caveats, limitations, refs)
```

### Key Features

1. **Consistent header** — Priority, Category, Prerequisites always in same place
2. **Status Checks** — Quick verification before you start
3. **Expected output** — Know what success looks like
4. **Common Issues** — Troubleshooting baked in
5. **Rollback** — Escape hatch for every change
6. **Related Playbooks** — Wikilinks for navigation
