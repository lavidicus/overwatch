# 2026-03-01 - Session Notes

## Issue Discovered: Context Window Roll Over Failure

**Time:** ~15:24 UTC  
**Severity:** P2 (High)

### What Happened

During session startup, discovered that llama-server on host `olla` was rejecting prompts exceeding 65536 tokens, even though SWA (sliding window attention) with `--context-shift` was enabled. Server logs showed:

```
error: request (65801 tokens) exceeds the available context size (65536 tokens)
```

### Root Cause

**Client-side context accumulation**, not server-side. OpenClaw was building prompts to 66k+ tokens before sending to server, exceeding the server's configured `--ctx-size 65535`.

SWA checkpoints were cycling fine (1→8→1), but the client didn't trigger compaction before the accumulated prompt exceeded the limit.

### Fixes Implemented

1. **Server ctx-size increase** (ready to deploy):
   - Created updated `llama-server.service` with `--ctx-size 131072`
   - Provides 2x buffer for prompt accumulation
   - See `DEPLOY-LLAMA-SERVER.md` for deployment instructions

2. **OpenClaw config update** ✅:
   - Updated `~/.openclaw/openclaw.json`
   - Changed `reserveTokens: 20000` → `reserveTokens: 40000`
   - Compaction now triggers at ~216k tokens (33% buffer)

3. **Code-level guard** (pending):
   - Prompt builder should check before sending: `if (tokens > ctx_size * 0.9) trigger_compaction()`
   - Requires OpenClaw source modification
