# Self-Improvement Program (SIP)

This directory tracks continuous-improvement ideas for Sam (the ops butler AI) and the OpenClaw workspace.

## Goals
- Capture ideas that make the assistant faster, safer, or more useful.
- Prioritize improvements and document outcomes.
- Keep a transparent history of what changed and why.

## Workflow
1. **Log the idea** in `sip/log.md` (date, description, expected impact, owner/status).
2. **Plan & execute** the improvement (reference related PRs/commits or tickets).
3. **Update the log** when complete, including measured impact if possible.
4. **Review regularly** (during heartbeats or when idle) to pull the next best improvement.

## Entry Template
```
## YYYY-MM-DD – <Title>
- **Idea:** What we want to change or explore.
- **Why:** Expected value / motivation.
- **Plan:** High-level steps or links to detailed docs.
- **Status:** proposed | in-progress | done | parked
- **Notes:** Observations, metrics, links, blockers.
```

Add new entries in reverse chronological order (newest at top) so the latest work is easy to find.
